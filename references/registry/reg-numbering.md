# 编号检查项注册表

> **唯一权威定义**：所有编号相关检查项在此定义，其他文件只能引用ID。
> **适用范围**：ALL = 所有类型 | S = Skill | P = Plugin | M = Memory

## 检查项列表

| ID | 检查项 | 要求 | 严重性 | 适用 |
|----|--------|------|--------|------|
| B-001 | 连续编号 | 无跳跃 (1,2,3 非 1,2,4) | Warning | ALL |
| B-002 | 格式一致 | 同一风格 (R1/R2 或 1.1/1.2) | Warning | ALL |
| B-003 | 引用存在 | 所有引用的 R1、Step 2 等已定义 | Severe | ALL |
| B-004 | 无重复编号 | 每个编号只用一次 | Warning | ALL |
| B-005 | 层级一致 | 1.1、1.2 在 Section 1 下 | Warning | ALL |
| B-006 | 跨节连续 | 如果编号跨节，无断点 | Warning | S,P |
| B-007 | 跨节区分 | Section A 的 1.1 ≠ Section B 的 1.1 (同范围时) | Severe | S,P |
| B-008 | 重置模式一致 | 新节的编号重置模式清晰 | Warning | S,P |

## 执行方法

```
1. 提取所有编号项：R1、Step 1、1.1 等
2. 检查连续性：是否有跳跃
3. 检查唯一性：是否有重复
4. 检查层级：子编号是否在正确父级下
5. 输出：「编号检查: N个编号项, M个问题」
```

## 常见问题模式

| 模式 | 示例 | 问题 |
|------|------|------|
| 跳跃 | 1, 2, 4 | 缺少 3 |
| 重复 | 两个 R5 | ID冲突 |
| 层级错误 | 1.1.1 在 Section 2 下 | 父级不匹配 |
| 过时引用 | 引用 R5 但 R5 已删除 | 断链 |
